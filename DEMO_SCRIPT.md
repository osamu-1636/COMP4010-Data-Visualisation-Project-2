# Demo Script — From Crisis to Corridor

## 0. Opening
This dashboard tells one focused story: forced migration begins as crisis pressure, grows into scale, lands unevenly across host geographies, forms origin-host corridors, and can be analyzed through country rankings and distance structure.

## 1. Scale
Use the KPI cards and trend chart to establish magnitude and historical context.

## 2. Host geography
Use one choropleth map to show where the selected cross-border stock is located. Emphasize that repeated host maps were removed to keep the story focused.

## 3. Corridors
Graph 1 answers where routes go. Graph 4 Sankey answers which origins, host countries and population statuses dominate the structure.

## 4. Rankings
Graph 5 and Graph 6 directly answer the two core ranking questions: top origin countries and top host countries.

## 5. Host concentration
The treemap complements Graph 6. Graph 6 gives exact rank order; the treemap shows whether hosting is dominated by one or two countries.

## 6. Distance analytics
This is the analytics layer: each origin-host corridor is assigned a distance band using centroid distance. It answers whether displacement stays near, remains regional, or expands far.

## 7. Crisis case
Use a selected crisis to turn the global pattern into one concrete humanitarian story.

## 8. Method
Close by explaining that the app reads cleaned/chart-ready outputs only and uses a modular Shiny architecture.
